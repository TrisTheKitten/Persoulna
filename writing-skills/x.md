# X Writing Pattern

X is for compressed ideas, strong claims, timely takes, public discussion, links, and threads. Do not spend the first sentence warming up.

## Adaptation

- Make the persona sharper and more economical.
- Put the strongest claim first.
- Remove soft setup phrases.
- Use one CTA at most.
- Use a thread only when each post adds value.
- Avoid generic hashtags.

## Length

- Single post: 120-240 characters preferred
- Thread: 2-6 posts
- One claim per post

## Patterns

Single post:

```text
[Strongest claim/result/tension]
[Specific proof or implication]
[One CTA if needed]
```

Thread:

```text
1. [Standalone hook with the full promise]
2. [Context or problem]
3. [Point 1]
4. [Point 2]
5. [Example/proof]
6. [Final takeaway or CTA]
```

Blog-to-X:

```text
1. [Most shareable claim from the blog]
2. [Why people should care]
3. [Useful detail]
4. [Memorable lesson]
5. [Link to full post]
```

## Check

- Strongest claim is first.
- Thread posts each add new value.
- Persona is sharper, not replaced.
- CTA is singular.
- Sensitive claims are not overstated.
